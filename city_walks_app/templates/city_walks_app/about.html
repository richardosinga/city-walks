Contributing · City Walks

How to add
a walk.

Every walk in this collection is a markdown file in a GitHub repository — a route, a list of stops, and prose that a person can actually read on the street. Adding a walk means writing that file. An AI agent can help with the research and the first draft; a human who knows the neighbourhood makes it worth reading.

1
Fork the repository on GitHub

The walk content lives at github.com/richardosinga/city-walks. Fork the repo, clone it locally, and create a branch for your walk.

# Clone and branch git clone https://github.com/YOUR_USERNAME/city-walks cd city-walks git checkout -b walk/amsterdam-oud-west

The walk content lives in content/, organised by continent → country → city. Create your walk file in the right city directory.

2
Write the walk file

A walk is a markdown file with a YAML frontmatter block. The required fields are type: walk, a route (list of [lat, lng] pairs), and a waypoints list of stop slugs.

# content/europe/netherlands/amsterdam/my_walk.md --- title: "A Walk Through the Harbour" type: walk image: amsterdam.jpg latitude: 52.3740 longitude: 4.8897 waypoints: - stop_one_slug - stop_two_slug - stop_three_slug route: - [52.3740, 4.8897] - [52.3745, 4.8903] - # ... more coordinates --- Walk prose goes here. Link to stops inline like [Stop One](/europe/netherlands/amsterdam/stop_one_slug). Each link becomes a tappable card on the detail page.
Route coordinates can be exported as a GPX from any mapping app, or traced in Google Maps. The walk renderer draws the polyline and computes distance automatically.

Each waypoint is either a POI file you create in the same directory, or a POI that already exists in the World66 content — the app falls back to World66 automatically, so there's no need to duplicate content that's already there.

3
Use an AI agent for the first draft

The prose and stop descriptions are where the work is. Claude Code — or any agent with access to the repo — can draft both, working from World66's existing neighbourhood and POI content as source material. The agent follows the same git workflow as a human contributor.

Write an architecture walk for Oud-West, Amsterdam. Use the POIs and neighbourhood descriptions already in the World66 content at content/europe/netherlands/amsterdam/. Create new POI files for stops that don't exist yet. The walk should tell an architectural story — not a list of sights, but a narrative a person can follow on foot.

The agent can also geocode waypoints (using the coordinates already in World66 POI files), generate the route from a rough sketch, and check that all linked slugs resolve to real files.

Agents work best when given a specific neighbourhood and a specific angle — "architecture", "food history", "1970s counterculture". A walk with no argument is harder to write than one that knows what it's trying to say.
4
Run locally and check the cards

Start the dev server and open your walk. The walk detail page splits the prose into per-stop cards — check that the section boundaries fall in the right places and that clicking a stop name opens the correct POI card.

# Install deps and run pip install -r requirements.txt python manage.py runserver 8067 # Open your walk open http://localhost:8067/europe/netherlands/amsterdam/my_walk

If a stop isn't linking correctly, check that the slug in the waypoints list matches the filename of the POI file, and that the inline link in the prose uses the full path.

5
Open a pull request

Push your branch and open a PR against main. Include a short note about the walk — the neighbourhood, the angle, any sources you used. Walks are reviewed for accuracy and prose quality before merging.

git add content/ git commit -m "add: Amsterdam Oud-West architecture walk" git push -u origin walk/amsterdam-oud-west gh pr create --title "Add: Amsterdam Oud-West architecture walk"
— ready to walk?
